# 12 — PRE-CODING RESEARCH PROTOCOL
## Step-by-Step Research Before Any Code

---

## THE RULE

**You MUST complete this protocol before writing ANY code.**

No exceptions. No shortcuts. No "I'll research as I go."

Research first. Code second. Always.

---

## THE 10-STEP PROTOCOL

### STEP 1: Identify the Project Type (2 minutes)

Read file `19-PROJECT-TYPE-TEMPLATES.md` and answer:
- What project is this? (Khemji / ShipBridge / MAAC / Silbar)
- What are the NEVER constraints? (never use X library)
- What techniques are assigned to this project?
- What's the signature interaction?

**Output:** Project type + constraints + technique budget

---

### STEP 2: Study Reference Sites (10 minutes)

Read file `13-AWWWARDS-REFERENCE-SITES.md` and pick 3-5 sites relevant to the section you're building.

For each site:
1. Open the site
2. Study the specific technique (not just "look at it")
3. Note: timing functions, easing curves, scroll triggers, color transitions, typography behavior
4. Describe what makes it unforgettable in 1-2 sentences

**Output:** 3-5 reference techniques with specifics

---

### STEP 3: Check Technique Inventory (5 minutes)

Read file `10-TECHNIQUE-INVENTORY-AND-RESEARCH-2026.md` and answer:
- Which techniques are available for this project?
- Which techniques are already used (don't repeat)?
- Which technique best fits this section?

**Output:** Selected technique(s) for this section

---

### STEP 4: Read GitHub Repos (5 minutes)

Read file `14-GITHUB-REPOS-BIBLE.md` and find:
- Is there a library for this technique?
- Does the project already have this library?
- If yes, check how it's used in the existing codebase
- If no, do we need it? (Check the decision framework in file 11)

**Output:** Library decision (use existing / add new / build custom)

---

### STEP 5: Watch YouTube Tutorials (10 minutes)

Read file `15-YOUTUBE-LEARNING-MAP.md` and find:
- Is there a tutorial for this technique?
- What's the approach shown?
- What can we adapt (not copy)?

**Output:** Technique approach + timing functions + implementation notes

---

### STEP 6: Read Code Patterns (5 minutes)

Read file `16-CODE-PATTERNS-LIBRARY.md` and find:
- Is there a copy-paste pattern for this technique?
- What's the base code we can start from?
- What modifications are needed?

**Output:** Base code pattern + modifications

---

### STEP 7: Read Design Philosophy (3 minutes)

Read file `01-DESIGN-REVOLUTION-BIBLE.md` and answer:
- What design principle applies here?
- What's the emotional goal of this section?
- What's the "one thing" this section should be remembered for?

**Output:** Design principle + emotional goal + "remember one thing"

---

### STEP 8: Check Project-Specific Files (5 minutes)

Check if there are project-specific research files:
- Khemji: Check existing Khemji components
- ShipBridge: Check ship-bridge-new components
- MAAC: Check maac-next components
- Silbar: Check silbar-security components

**Output:** Existing patterns to follow + gaps to fill

---

### STEP 9: Write the Plan (3 minutes)

Before writing code, write a 3-line plan:
```
SECTION: [section name]
TECHNIQUE: [technique to use]
REFERENCE: [reference site]
CONSTRAINTS: [what NOT to use]
OUTPUT: [what the user will see/feel]
```

**Output:** Clear plan before coding

---

### STEP 10: Now Write Code

Only after completing steps 1-9, you may write code.

---

## THE CHECKLIST

Before writing ANY code, confirm:

- [ ] I know the project type and constraints
- [ ] I studied 3-5 reference sites
- [ ] I checked the technique inventory
- [ ] I checked GitHub repos
- [ ] I watched relevant YouTube tutorials
- [ ] I read code patterns
- [ ] I understand the design philosophy
- [ ] I checked existing project files
- [ ] I wrote a 3-line plan
- [ ] I know what the user will see/feel

**If ANY box is unchecked: go back and complete that step.**

---

## THE TIME BOX

This protocol should take 30-45 minutes total.

If it takes longer, you're over-researching. Pick the top 3 reference sites, skip YouTube, and start coding.

**The goal is informed coding, not exhaustive research.**

---

## THE EXCEPTIONS

You may skip this protocol IF:
1. The task is a trivial fix (typo, color change, spacing adjustment)
2. You've already researched this exact technique in the last 24 hours
3. The user explicitly says "just code it, don't research"

In all other cases: **complete the protocol.**

---

## HOW TO USE THIS FILE

When you start ANY task:
1. Read this file
2. Follow the 10 steps
3. Check the checklist
4. Write the plan
5. Then write code

**Research is not optional. It's the difference between generic and unforgettable.**

## Step-by-Step Deployment Guide

### Step 1: Create Missing Public Assets

Before pushing to Git, create these files in `public/`:

1. **`public/favicon.ico`** — 32x32 or 16x16 shield/security icon
2. **`public/apple-touch-icon.png`** — 180x180px PNG
3. **`public/logo.png`** — Recommended 512x512px, used in Organization schema
4. **`public/og-image.jpg`** — 1200x630px, used for social sharing and AI search

For the OG image, create a branded image with:
- Dark background (#0a0a0a)
- "Silbar Security Services" text
- Gold accent (#BF953F)
- Tagline: "PSARA-Licensed | ISO 9001:2015 | 7,000+ Officers"
- Website URL at bottom

### Step 2: Optimize hero.mp4

14MB is large. Options:
- Compress with HandBrake (target: 3-5MB)
- Or use a service like Cloudinary for video optimization
- Or serve from a CDN (Bunny.net, Cloudflare Stream)

If keeping as-is, Vercel will serve it from its CDN automatically.

### Step 3: Create `vercel.json` (Optional but Recommended)

```json
{
  "framework": "nextjs",
  "regions": ["bom1"]
}
```

`bom1` = Mumbai region — closer to your Indian audience than the default US region.

### Step 4: Push to GitHub

```bash
git add .
git commit -m "Production-ready: add public assets for Vercel deployment"
git push origin main
```

### Step 5: Import to Vercel

1. Go to [vercel.com/new](https://vercel.com/new)
2. Click **"Import Git Repository"**
3. Select your repository
4. Vercel auto-detects Next.js and sets:
   - Framework: Next.js
   - Build Command: `next build`
   - Output Directory: `.next`
   - Install Command: `npm install`

### Step 6: Configure Environment Variables

In Vercel Dashboard → Project Settings → Environment Variables:

| Variable | Value | Environment |
|----------|-------|-------------|
| `GOOGLE_VERIFICATION` | Your Google Search Console verification code | Production |

That's it — no other env vars needed. The codebase has zero `process.env` references in source code.

### Step 7: Configure Custom Domain

1. In Vercel Dashboard → **Domains**
2. Add `www.silbarsecurity.in`
3. Add `silbarsecurity.in` (set as redirect to www)

**DNS Records** (at your domain registrar):

| Type | Name | Value |
|------|------|-------|
| `A` | `@` | `76.76.21.21` |
| `CNAME` | `www` | `cname.vercel-dns.com` |

SSL is automatic via Let's Encrypt.

### Step 8: Deploy

Click **Deploy**. Vercel will:
1. Clone repo (shallow clone)
2. Install dependencies
3. Run `next build` (generates ~280 static pages)
4. Deploy to global CDN

## Build Considerations

### Page Count Impact
- ~280 static pages generated at build time
- Hobby tier: 15-minute build timeout
- Pro tier: 30-minute build timeout
- If build is slow, ISR can reduce pre-rendered pages

### GSAP on Vercel
- Your `src/lib/gsap.ts` has the `typeof window` guard — SSR-safe
- All GSAP components use `'use client'` — correct
- No build issues expected

### Lenis on Vercel
- `src/lib/lenis.ts` is only called from `HomePageClient.tsx` (client component)
- SSR-safe, no build issues

### ISR Configuration
- State pages: `revalidate = 86400` (daily) — already set
- City pages: `generateStaticParams()` — static at build, consider adding `revalidate = 86400`
- Blog posts: static at build — fine as-is

## Post-Deployment Checklist

1. **Build logs** — Check for errors in Vercel dashboard
2. **Preview deployment** — Visit every page type:
   - Homepage with hero video
   - Service detail page
   - Industry detail page
   - Blog post
   - State SEO page
   - City SEO page
   - Contact page
3. **GSAP animations** — Verify ScrollTrigger works on all pages
4. **Lenis smooth scroll** — Test on homepage (desktop + mobile)
5. **Video playback** — Verify hero.mp4 loads and plays
6. **Sitemap** — Check `https://www.silbarsecurity.in/sitemap.xml`
7. **Robots.txt** — Check `https://www.silbarsecurity.in/robots.txt`
8. **OG image** — Test with [Facebook Debugger](https://developers.facebook.com/tools/debug/)
9. **SSL** — Verify HTTPS works on both www and non-www
10. **Lighthouse** — Run audit (target: 90+ performance, 100 accessibility)
11. **Mobile** — Test at 320px, 480px, 768px widths
12. **Search Console** — Submit sitemap, verify domain ownership

## Vercel Analytics (Optional, Post-Launch)

```typescript
// Inside <body>:
<Analytics />
<SpeedInsights />
```

Then enable in Vercel Dashboard → Analytics / Speed Insights.
